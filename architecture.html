Problem → Solution → Result
01
Problem

원본 이미지(최대 44MB)를 그대로 저장해 페이지 로딩이 극도로 느리고, Next.js 이미지 캐시가 LRUCache: calculateSize returned 0 에러를 반복 발생시켰다.

Solution

업로드 시점에 sharp로 WebP 변환 + 최대 1920px 리사이즈(quality 80)를 적용하고, Cloudflare R2에 저장하도록 파이프라인을 구성했다.

Result

파일 크기 약 95% 감소(44MB → ~300KB), 이미지 캐시 에러 제거, 페이지 초기 로딩 속도 대폭 개선.

02
Problem

npm 패키지 재설치 후 Apple OAuth 프로바이더가 크리덴셜 없이 초기화되면서 proxy.ts가 /admin 경로 처리 중 무응답 상태가 됐다.

Solution

auth.config.ts에서 미설정 Apple 프로바이더를 제거하고 Google OAuth만 유지했다.

Result

관리자 페이지 정상 접근 복구, 불필요한 프로바이더 제거로 인증 설정 단순화.

03
Problem

일반 유저와 관리자 모두 같은 인증 흐름을 사용하면 권한 분리가 어렵고 보안 취약점이 생길 수 있다.

Solution

일반 유저는 Google OAuth + Credentials(이메일/비밀번호), 관리자는 별도 Admin 테이블과 ID/PW Credentials로 인증을 분리했다. proxy.ts에서 /admin 경로를 별도 보호.

Result

유저/관리자 권한 명확히 분리, 역할(master/staff) 기반 접근 제어 가능.
